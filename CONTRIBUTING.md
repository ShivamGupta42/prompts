# Contributing

Edits to existing prompts and proposals for new ones are both welcome. Read this once, then go straight to the workflow at the bottom.

## What this repo is, technically

A published mirror of an espanso text-expansion config. The source of truth lives in a private dotfiles repo; a small sync script extracts each prompt-style trigger and writes it here. That means:

- This repo is **read-mostly**. The files in `planning/`, `dev/`, `ux/`, `qa/`, `marketing/`, `learning/` and the cross-cutting `validation-criteria.md` are regenerated by the sync script — manual edits to them get overwritten on the next sync.
- The **safe place to edit a prompt** is the upstream espanso config (or, if you don't have it, propose the edit via PR and it'll be merged into the source and re-synced).
- `README.md`, `CONTRIBUTING.md`, `LICENSE`, `INDEX.md`, `safety/`, and `.github/` are **not synced** — those are owned by this repo directly.

## The safety stack

This is a public repo, so the surface is permanent. Three independent layers gate every commit:

1. **Local pre-commit hook** — `gitleaks` (catches API keys, tokens, AWS/GCP creds, JWT, PEM blocks) + a custom regex hook (catches personal-info patterns specific to the repo owner) + `detect-private-key`.
2. **GitHub Action** — server-side mirror of the same scans on every PR. Catches commits that bypassed the local hook (e.g. via `--no-verify`).
3. **Branch protection on master** — no direct push, PR required, the `scan` status check must pass before merge. Force pushes and deletions are blocked. Enforced on admins.

The bait test that proved each layer works is documented in [safety/PROVEN.md](safety/PROVEN.md). The custom patterns the local hook blocks live in [safety/patterns.txt](safety/patterns.txt) — one Python regex per line. If you fork this repo, edit `safety/patterns.txt` to match what you want to keep out of *your* mirror.

## Workflow

```sh
git clone git@github.com:ShivamGupta42/prompts.git
cd prompts

# One-time setup
brew install pre-commit gitleaks
pre-commit install

# Make your change on a feature branch
git checkout -b proposal/short-name
# … edit safety/patterns.txt or any non-synced file …
git add <file>
git commit -m "..."     # local hook runs automatically; fix anything it flags
git push -u origin proposal/short-name
gh pr create
```

The `safety-scan` GitHub Action will run on the PR. Once it passes and the PR has the OK, it can be merged.

## Proposing a new prompt or editing a synced one

Synced prompts (everything under `planning/`, `dev/`, `ux/`, `qa/`, `marketing/`, `learning/`, plus `validation-criteria.md` at root) get overwritten on every sync. To propose a change:

- **Open an issue** describing the change you want and the trigger it'd modify.
- The owner will edit the upstream espanso config and re-sync, which lands the change here.

This indirection exists because the espanso config is the canonical source and propagating from one place to two would create drift.
