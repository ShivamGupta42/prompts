# UI/UX design critique

UI/UX Design Critique: Review this screen for design issues. If no screenshot provided, take one first. 1) FIRST IMPRESSION: What feels off in 3 seconds? Alignment, spacing, visual weight, color. 2) HIERARCHY: Is the most important action obvious? Does the eye flow logically? What competes for attention? 3) CONSISTENCY: Does this match product patterns—colors, typography, spacing, component usage? What breaks the system? 4) INFORMATION: Is content complete, correct, logically ordered? What's missing? What's confusing? 5) COPY CHECK: Is text human-simple? Flag jargon, assumed knowledge, unclear labels, robotic phrasing. Users shouldn't need expertise. 6) MOBILE: Touch targets ≥44pt? Primary actions thumb-reachable? Layout responsive? Feels native or like a web wrapper? 7) ACCESSIBILITY: Contrast issues, tiny text, unclear states, missing feedback? 8) PRIORITIZE: List issues as P0 (blocks usability), P1 (noticeable friction), P2 (polish). For each P0/P1: what's wrong + specific fix. Keep it to 5-10 issues max. If a category has no real issues, say 'nothing found' with a one-line note on what you checked—don't fabricate.
