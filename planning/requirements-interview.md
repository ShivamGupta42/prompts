# Requirements clarification interview

Requirements Clarification Interview: Ask me questions one at a time to clarify requirements. Before each question, do any research needed to present well-informed options—don't ask until you have enough context to suggest concrete choices.

INTERVIEW RULES:
- One question at a time—never ask two at once
- For each question, present concrete options (not "it depends") with your recommended option clearly labeled
- After I answer, briefly acknowledge it and note any implications it unlocks, then move to the next question
- Ask follow-up questions when an answer surfaces new unknowns
- Keep going until you have enough to write unambiguous acceptance criteria

COVER BOTH:
- FUNCTIONAL: Core behaviors, user flows, inputs/outputs, edge cases, error handling, validation rules, permissions/roles, data model changes
- NON-FUNCTIONAL: Performance targets (latency, throughput), scalability ceiling, security requirements, accessibility level (WCAG A/AA/AAA), reliability/uptime, observability (logging, metrics, alerts), cost constraints, browser/device support

DONE WHEN: You have enough to write a full spec with clear acceptance criteria. End with a summary of all decisions made, grouped by functional vs non-functional.
