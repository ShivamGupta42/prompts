# Happy-path smoke test

Quick Smoke Test: Generate a minimal happy-path QA checklist for this feature. Focus ONLY on the golden path—what 80% of users will do. 1) SETUP: One-liner to get into the right state. 2) STEPS: Numbered list of exact actions (5-10 max). Use real values, not placeholders. 3) EXPECTED: What success looks like at each step—be specific. 4) DONE WHEN: The single observable outcome that proves it works. Skip edge cases, accessibility, cross-browser—this is just 'does the happy path work?' Output as a quick checklist I can run in under 2 minutes.
