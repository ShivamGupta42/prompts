# Generate manual QA checklist

Manual QA Checklist: Generate a comprehensive manual testing checklist for this feature branch. 1) PREREQUISITES: List exact environment setup—server state, database seeds, test accounts, feature flags, env vars. What must be true before testing starts? 2) TEST DATA SETUP: Exact commands or steps to create the required test state from step 1. Include cleanup steps. (This is the IMPLEMENTATION of step 1's prerequisites — it must come BEFORE the test procedures that depend on the state existing.) 3) FEATURE INVENTORY: Use git diff against main to identify all user-facing changes. Group by: new features, modified behaviors, removed functionality. 4) TEST PROCEDURES: For each change, provide step-by-step actions—exact clicks, inputs, navigation. Include specific test data values, not placeholders. What does the user see at each step? 5) EXPECTED OUTCOMES: Per step—what success looks like (UI state, data changes, network calls). Be precise: 'Toast appears within 2s' not 'feedback shown'. 6) EDGE CASES: Per feature—empty states, boundary values, invalid input, rapid repeated actions, interrupted flows, concurrent operations, network failures, permission denied. 7) ACCESSIBILITY: Keyboard-only navigation path, focus management, screen reader announcements, color contrast, touch targets (44px min). 8) CROSS-BROWSER: Note any features needing Safari/Firefox/mobile verification. 9) REGRESSION RISKS: What existing features might break? List specific flows to verify unchanged. 10) PRIORITY: Mark each test P0 (blocks release), P1 (should test), P2 (nice to have). Output as executable checklist with [ ] checkboxes.
